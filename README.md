# swan-planner · 空地异构集群任务规划系统

基于大模型的**空地异构集群(无人机 + 无人车)分层任务规划系统**指挥控制台。
采用「大模型做慢思考、经典算法做快反应」的三层规划架构,用 **PySide6(Qt)** 实现。

![界面预览](docs/preview.png)

## 分层架构

| 层级 | 职责 | 所用模型 | 输出 |
|------|------|---------|------|
| **L1 顶层** | 全局任务分解、集群编组、组任务分配 | `Qwen3.6-27B` | 分组—任务方案 |
| **L2 中层** | 组内任务细化、平台角色分配、局部重规划 | `Qwen3.6-27B` | 各平台行为序列 |
| **L3 底层** | 语义指令→技能原语、视觉理解、执行监控 | `Qwen2.5-7B + VL-7B` | 技能原语调用 |

设计要点:大模型只做规划、**不进控制环**;运动/避障由经典控制栈执行;
场景以结构化 JSON 描述作为大模型与物理世界之间的接口;层间以统一 Schema 传递任务。

> 详细设计思路见 [`docs/design.md`](docs/design.md)。

## 规划管线(后端)

规划器以 **JSON 场景描述**([`data/scene.json`](swan_planner/data/scene.json))与
**平台能力档案**([`data/platforms.json`](swan_planner/data/platforms.json))为输入,
通过**能力匹配**完成任务分配:

1. **解析指令**(L1)—— 自然语言 → 目标列表(侦察 / 运输)。生产环境由 Qwen 完成。
2. **场景落地**(L1)—— 读取 JSON 场景,提取禁飞天花板、目标置信度、障碍与通信盲区。
3. **构建任务规格** —— 每个任务角色声明 `required` / `preferred` 能力与载荷需求
   (侦察需 `aerial_survey`;运输需 `transport` 且载荷达标 + `comm_relay` 护航)。
4. **能力匹配分配** —— 对候选平台做可行性过滤 + 打分(能力契合 + 就近 + 电量/续航),
   择优指派;能力/载荷不满足者自动编入预备组。
5. **分组细化**(L2)—— 侦察按区域面积与扫描带宽**分解扇区**、由建筑高度推导扫描高度;
   运输在地面通行子图上做 **Dijkstra 路径搜索并规避障碍**、由禁飞天花板推导护航高度。
6. **技能编译**(L3)—— 译为带参数的技能原语(`goto/scan/vl_detect/follow/...`)。

**执行逻辑优化**:L1(全局规划,含分配)在后台线程只跑一次;选组仅做 L2/L3 的
惰性组装(纯查询,毫秒级),不重算 L1。

当前大模型 I/O 以**预测实例**驱动上述确定性算法,接口与真实推理服务一致,可无缝替换。

## 界面构成

- **头部栏** —— 任务状态、作战时钟、告警、人在回路审核开关
- **左栏 · 集群编组** —— L1 输出的分组结果,空(青)/地(琥珀)双色编码
- **中央 · 态势总览** —— 区域 / 平台 / 路径 / 场景图,可在「态势 / 路径规划 / 场景图」间切换
- **右栏 · 任务下达 + 分层推理链** —— 自然语言下达,展示 L1→L2→L3 完整推理链
- **底部 · 执行时间线** —— 三层活动的甘特图

## 运行

```bash
pip install -r requirements.txt
python main.py
```

> 需要图形界面环境。当前大模型 I/O 由 `swan_planner/models/mock_llm.py` 以**预测实例**模拟,
> 接口与真实推理服务保持一致,后续可无缝替换。

## 工程结构

```
swan-planner/
├── main.py                     # 入口
├── requirements.txt
└── swan_planner/
    ├── app.py                  # 主窗口装配与信号接线(L1 一次 / L2-L3 按组)
    ├── config.py               # 配色 / 模型命名 / 层级定义
    ├── theme.py                # 全局 QSS 主题
    ├── data/
    │   ├── scene.json          # JSON 场景描述(区域/物体/约束/通行边)
    │   └── platforms.json      # 平台能力档案
    ├── models/
    │   ├── scene.py            # 场景解析 · 几何查询 · 路径搜索 · LLM 上下文
    │   ├── data.py             # 领域对象与 JSON 加载器
    │   └── planner.py          # 规划引擎:能力匹配任务分配 + 分层推理
    └── widgets/
        ├── header_bar.py       # 头部栏
        ├── group_tree.py       # 集群编组
        ├── situation_map.py    # 态势地图(QGraphicsView)
        ├── task_panel.py       # 任务下达
        ├── reasoning_chain.py  # 分层推理链
        └── timeline.py         # 执行时间线
```
